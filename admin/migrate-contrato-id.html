Migración · asignar contrato_id

Script de migración N2 del refactor multi-contrato. Asigna el campo contrato_id a los documentos legacy que aún no lo tienen. Idempotente: re-ejecutarlo no modifica los docs ya migrados.

⚠ Antes de ejecutar: esta operación marca todos los docs sin contrato_id con el valor que indiques abajo. Si solo hay un contrato vigente (4123000081), todos los 228+ docs existentes recibirán ese valor. Si por alguna razón hay docs de otros contratos mezclados, debes limpiarlos primero.
